# Disclaimer and Authorized Use Notice

ICIT AttackSimPro is a **Purple Team security validation product** designed to perform
**authorized, non-destructive security simulations** for the purpose of validating
security controls, logging, detection, and response capabilities.

This product is not a penetration testing platform and is not intended to perform
covert, exploitative, or adversarial red-team operations.

---

## Authorized Use Only

AttackSimPro may only be used when **explicit written authorization** has been granted
by the system owner. All activity must be:

- Limited to explicitly approved targets
- Executed within an agreed scope and time window
- Coordinated with relevant stakeholders
- Conducted in a Purple Team (collaborative) model

Use of AttackSimPro against systems without authorization is strictly prohibited.

---

## Prohibited Activities

The following activities are explicitly out of scope and must not be performed using
AttackSimPro:

- Exploit development or exploit payload delivery
- Credential harvesting, brute-force, or password spraying
- Malware deployment or command-and-control infrastructure
- Persistence mechanisms or covert access techniques
- Data exfiltration, data manipulation, or destructive testing
- Lateral movement beyond explicitly approved scope

If a test requires any of the above activities, it is out of scope for AttackSimPro.

## Compliance Alignment

AttackSimPro supports security validation activities aligned with common frameworks,
including but not limited to:

- PCI DSS (testing and monitoring validation)
- SOC 2 (CC7.x detection and response)
- NIST Cybersecurity Framework (DE.CM, RS.AN)

Use of AttackSimPro does not constitute certification, attestation, or formal compliance
assessment.
